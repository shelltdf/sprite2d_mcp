# 软件设计

## 总体结构

- **表现层**：Vue 3 单页应用，组件化拆分菜单栏、工具栏、左中右主区、状态栏。
- **状态层**：Pinia 存储工程与精灵列表、当前选中 ID、画布视图变换（缩放/平移）。
- **画布层**：HTML5 Canvas 2D 渲染图集占位背景、精灵矩形、选中高亮。
- **编译层**：纯函数将精灵布局合并为导出描述（JSON）并触发浏览器侧 PNG 导出（Canvas `toBlob`）。

## 模块划分

| 模块 | 职责 |
|------|------|
| `App` / 布局壳 | 网格分区，挂载各面板 |
| 菜单/工具栏 | 命令分发，调用 store 与导出 |
| 精灵列表 | 展示 `sprites[]`，点击选中 |
| 画布 | 根据 store 绘制，鼠标交互（平移/缩放/点选） |
| 属性面板 | 双向绑定选中精灵字段（受控校验后续可增强） |
| 编译导出 | 生成 `atlas.json` 与合成图（最小可行：轴对齐装箱占位） |

## 与 `foo`/helloworld 说明

本仓库产品为精灵表工具，**不**采用 init 规则中的 helloworld + foo 示例；物理阶段以 `sprite-sheet-compiler-gui` 目标为准。
